# 升級規則與人機協作

## 基本資訊

| 欄位 | 值 |
|------|-----|
| 文件名稱 | escalation_rules |
| 模組歸屬 | module_04_customer_service |
| 用途 | 定義 AI 自動處理與人工接手的邊界、升級流程、通知機制 |

---

## 設計原則

> 客服不是越自動越好，重點是「該自動的自動、該人工的人工」。

- AI 處理重複性高、風險低的問題
- 涉及承諾、金錢、情緒的事情交給人
- 升級要快、要順暢，客戶不應感到被踢皮球
- 每次升級都帶完整上下文，人工接手不需要重新問

---

## 自動處理範圍

以下類型的問題由 AI 直接回覆，不需人工介入：

| 類別 | 範例 |
|------|------|
| 營業資訊 | 營業時間、地址、聯繫方式 |
| 服務項目 | 提供哪些服務、服務內容簡介 |
| 一般流程 | 合作流程、報名流程、付款方式（不含具體金額） |
| FAQ 問題 | 知識庫中已有明確答案的問題 |
| 導引指示 | 引導客戶到正確的頁面、表單、聯繫管道 |
| 簡單確認 | 「有收到您的訊息」「已記錄您的需求」 |

### AI 自動回覆的品質標準

- 回覆時間：< 5 秒
- 答案來源：100% 來自知識庫，零編造
- 語氣：友善、專業、簡潔
- 結尾：主動詢問「還有其他問題嗎？」或提供下一步建議

## 必須轉人工的情境

以下情境 AI **必須**升級轉人工，不可自行回覆：

### 高優先級（立即升級）

| 情境 | 判斷依據 | 原因 |
|------|---------|------|
| 報價討論 | 客戶詢問「多少錢」「費用」「價格」 | 價格需要根據需求客製，AI 不可承諾 |
| 合約相關 | 客戶提到「合約」「簽約」「條款」 | 法律文件需人工處理 |
| 金額討論 | 任何涉及具體金額的對話 | 財務承諾需授權人員確認 |
| 時程承諾 | 客戶要求確認交付時間 | 時程需評估後才能承諾 |
| 客訴處理 | 客戶表達不滿、抱怨、投訴 | 情緒問題需要人的同理心 |
| 退款請求 | 客戶要求退款或取消 | 財務操作需人工授權 |

### 中優先級（盡快升級）

| 情境 | 判斷依據 | 原因 |
|------|---------|------|
| 技術故障 | 客戶回報產品或服務異常 | 需要技術團隊排查 |
| 複雜需求 | 客戶需求超出標準服務範圍 | 需要人工評估可行性 |
| 法律問題 | 涉及法律、責任、賠償 | 需法務或管理層處理 |
| 個資相關 | 客戶要求查看、修改、刪除個人資料 | GDPR / 個資法合規需求 |

### AI 判斷關鍵字（參考）

```
立即升級關鍵字：
- 價格、費用、報價、多少錢、收費、預算
- 合約、簽約、合同、條款、協議
- 退款、退費、取消、退貨
- 投訴、不滿、差勁、爛、騙

盡快升級關鍵字：
- 故障、壞了、不能用、出錯、bug
- 律師、法律、賠償、責任
- 個資、隱私、刪除我的資料
```

> 注意：關鍵字只是輔助，AI 應理解上下文語意，不是純字串匹配。「你們的定價策略是什麼」也應該轉人工，即使沒有出現「多少錢」。

## 升級流程

### 完整流程圖

```
AI 判斷需要升級
       ↓
Step 1: 回覆客戶
       「感謝您的詢問！這個問題讓我幫您轉接專人，
        請稍候，同仁會儘快與您聯繫。」
       ↓
Step 2: 發送通知給負責人
       包含：對話摘要 / 客戶資訊 / 建議優先級 / 升級原因
       ↓
Step 3: 寫入 CRM
       狀態標為「需人工處理」
       AI 互動記錄追加升級事件
       ↓
Step 4: 持續監聽
       若客戶在等待期間繼續發訊息，
       AI 回覆「已收到，專人正在處理中」
       不嘗試自行回答升級範圍的問題
       ↓
Step 5: 人工接手
       負責人直接聯繫客戶
       CRM 狀態更新為「跟進中」
```

### Step 1：客戶回覆範本

**標準升級回覆**：
```
感謝您的詢問！這個問題需要由專人為您處理，
我已經將您的需求轉達給負責同仁，將會儘快與您聯繫。

如有任何補充資訊，歡迎在這裡留言，我會一併轉達。
```

**客訴情境的升級回覆**：
```
非常抱歉造成您的困擾，我完全理解您的感受。
我已經將這個問題緊急轉達給負責同仁，
會以最快速度與您聯繫處理。

感謝您的耐心，我們會認真對待這個問題。
```

**非工作時間的升級回覆**：
```
感謝您的訊息！目前為非服務時間（服務時間：週一至週五 09:00-18:00）。
我已記錄您的需求，負責同仁將於下個工作日優先處理。

如有緊急需求，請撥打 [緊急聯繫電話]。
```

### Step 2：通知格式

發送給負責人的升級通知：

```
--- 客服升級通知 ---

升級原因：[報價討論 / 客訴 / 技術故障 / ...]
建議優先級：[高 / 中]
時間：2026-03-10 14:30

客戶資訊：
- 名稱：王大明
- 聯繫方式：@wang_dm (Telegram)
- 來源：Telegram DM
- CRM 狀態：跟進中 → 需人工處理

對話摘要：
客戶詢問企業內訓報價，團隊 15 人，
預計 Q2 執行，預算範圍 5-8 萬。
已多次互動，需求明確度高。

最近 3 則對話：
[客戶] 想了解 15 人內訓的費用
[AI] 感謝詢問！報價需根據您的具體需求客製，已轉專人處理。
[客戶] 好的，希望能盡快收到報價

建議動作：準備內訓提案與報價單
```

### Step 3：CRM 記錄

升級事件寫入 AI 互動記錄：
```
---
[2026-03-10 14:30] 升級事件
原因：報價討論
AI 判斷：客戶詢問具體費用，超出自動回覆範圍
動作：已通知 @sunny，已回覆客戶等待訊息
狀態：需人工處理
---
```

## 通知方式

| 管道 | 設定 | 適用情境 |
|------|------|---------|
| Telegram 私訊 | 發送到指定負責人的 Telegram | 主要通知管道，即時性高 |
| Email | 發送到指定 Email | 備用管道，或需要留存記錄時 |

### 通知設定建議

```yaml
notification:
  primary: telegram
  fallback: email
  recipients:
    default: "@sunny"  # 預設負責人
    complaint: "@manager"  # 客訴升級給主管
    technical: "@tech_lead"  # 技術問題升級給技術主管
  quiet_hours:
    enabled: false  # 客服升級不設靜音，重要通知隨時送
```

## 回應時間目標

| 場景 | 目標時間 | 說明 |
|------|---------|------|
| AI 自動回覆 | < 5 秒 | 知識庫匹配後立即回覆 |
| 升級通知發送 | < 10 秒 | AI 判斷升級後立即通知 |
| 人工接手（工作時間） | < 30 分鐘 | 負責人收到通知後開始處理 |
| 人工接手（非工作時間） | 下個工作日 09:30 前 | 上班後優先處理前一晚的升級案件 |

### 逾時處理

```
人工接手逾時 30 分鐘：
  → AI 對客戶發送：「專人正在處理中，感謝您的耐心等待」
  → 再次通知負責人

人工接手逾時 2 小時：
  → 升級通知發送給主管
  → AI 對客戶發送：「已為您優先安排處理，將儘快回覆」

人工接手逾時 4 小時（工作時間內）：
  → 標記為 SLA 違規
  → 通知所有相關人員
```

## 非工作時間處理

### 定義

```
工作時間：週一至週五 09:00-18:00 (UTC+8)
非工作時間：以上時段以外（含週末、國定假日）
```

### 非工作時間的 AI 行為

| 問題類型 | AI 處理方式 |
|---------|------------|
| FAQ 問題 | 正常自動回覆（知識庫不休息） |
| 需升級問題 | 回覆「已收到，人工將於上班時間回覆」，通知仍然發送 |
| 緊急問題（客訴、故障） | 回覆已收到 + 提供緊急聯繫方式，通知立即發送 |

### 非工作時間標準回覆

```
感謝您的訊息！目前為非服務時間。

服務時間：週一至週五 09:00-18:00（台灣時間）

我已記錄您的需求，負責同仁將於下個工作日優先與您聯繫。
如有緊急需求，請撥打：[緊急電話]

在等待期間，以下資訊可能對您有幫助：
[根據客戶問題提供相關 FAQ 連結]
```

## 人機協作最佳實踐

### AI 做得好的

- 24/7 即時回應，客戶不需要等
- 重複性問題回答品質一致
- 完整記錄每次對話
- 自動分類與排序優先級
- 永不遺漏、永不疲倦

### 人做得好的

- 理解複雜、模糊的需求
- 處理情緒性對話（投訴、不滿）
- 做出承諾與決策（報價、合約）
- 建立信任關係
- 處理例外情況

### 協作模式

```
最佳分工：
AI → 第一線接觸、資訊蒐集、FAQ 回覆、記錄、通知
人 → 決策、承諾、情緒處理、複雜問題、關係維護

不好的分工：
AI 嘗試處理所有事情 → 客戶體驗差
人處理所有事情 → 效率低、容易遺漏
AI 和人同時回覆 → 客戶困惑
```

## 課堂提示

- 升級規則寧嚴勿鬆：第一版多轉幾次人工沒關係，少轉了造成客戶體驗差才麻煩
- 上線第一週每天審視升級記錄：哪些不該轉的轉了？哪些該轉的沒轉？
- 最常見的錯誤：讓 AI 回答「大概 X 萬左右」—— 這就是報價，必須轉人工
- 升級通知的「對話摘要」品質很重要，人工接手時不需要從頭看對話
- 非工作時間不代表不通知，通知照發，只是回覆客戶的措辭不同
