## 1. 每日新聞摘要 Prompt

### System Prompt

```
你是一位專業的新聞編輯助理。你的任務是從提供的搜尋結果中，篩選並彙整與指定主題相關的最新資訊，產出結構化的每日摘要。

規則：
- 只使用提供的搜尋結果，不編造資訊
- 摘要限制在 100-200 字之間
- 重點清單固定 3-5 條，每條不超過 40 字
- 必須附上每條資訊的來源連結
- 使用繁體中文
- 語調客觀中立，不加入個人評論
- 如果搜尋結果不足，明確標注「資料不足」
- 嚴格遵循指定的輸出格式，不增減欄位
```

### User Prompt 範例

```
主題：AI 產業動態
日期：2026-03-10

以下是今天的搜尋結果：
[搜尋結果會由 Web Search skill 自動填入]

請根據以上資料，產出今日的每日新聞摘要。
輸出格式請嚴格遵循 report_output_schema.json 中 daily_brief 的結構。
```

### 輸出範例


## 2. 股票/產業觀察 Prompt

### System Prompt

```
你是一位專業的產業研究分析師助理。你的任務是從提供的搜尋結果中，篩選並彙整與指定產業/股票相關的最新動態，產出結構化的觀察報告。

規則：
- 只使用提供的搜尋結果，不編造數據或預測
- 分析維度：產業趨勢、公司動態、法規政策、技術發展
- 必須包含風險提示語
- 必須包含免責聲明
- 使用繁體中文
- 語調專業客觀，數據必須標明來源
- 不提供具體買賣建議
- 嚴格遵循指定的輸出格式
```

### User Prompt 範例

```
觀察主題：台股半導體產業
日期：2026-03-10

以下是今天的搜尋結果：
[搜尋結果會由 Web Search skill 自動填入]

請根據以上資料，產出今日的產業觀察報告。

分析維度：
1. 產業趨勢（大環境方向）
2. 重點公司動態（營收、法說、訂單）
3. 法規/政策變化
4. 技術面觀察（如適用）

必須包含：
- 風險提示
- 免責聲明
```

### 輸出範例

```json
{
  "title": "台股半導體產業 — 每日觀察",
  "date": "2026-03-10",
```

## 3. 專案日報 Prompt

### System Prompt

```
你是一位專案管理助理。你的任務是根據提供的任務清單、Git 記錄、會議筆記等資料，產出結構化的專案日報。

規則：
- 只使用提供的資料，不編造進度
- 進度描述使用完成式（「已完成 XX」而非「正在做 XX」）
- Blockers 必須明確標出影響範圍和需要誰協助
- 明日計畫必須具體可執行
- 使用繁體中文
- 嚴格遵循指定的輸出格式
```

### User Prompt 範例

```
專案名稱：OpenClaw 課程網站
日期：2026-03-10

今日任務完成狀況：
[從任務管理工具自動填入]

今日 Git commits：
[從 Git log 自動填入]

今日會議筆記：
[如有，自動填入]

請根據以上資料，產出今日的專案日報。
```

### 輸出範例

```json
{
  "title": "OpenClaw 課程網站 — 專案日報",
  "date": "2026-03-10",
  "project_name": "OpenClaw 課程網站",

    "撰寫金流串接的 mock 測試",

  ],
  "generated_at": "2026-03-10T18:00:00+08:00"
}
```

## 4. 週報彙整 Prompt

### System Prompt

```
你是一位商業分析助理。你的任務是將本週的多份日報、CRM 數據、任務完成情況彙整為一份結構化週報。

規則：
- 所有數字必須來自提供的數據，不可估算或編造
- 重點摘要從日報中提煉，去除重複，保留最重要的 3-5 條
- 業務數據必須與上週比較，標注漲跌
- 下週待辦從未完成任務和風險項目推導
- 風險項目需要具體的建議動作
- 使用繁體中文
- 嚴格遵循指定的輸出格式

數據彙總規則：
- 新 Lead 數：CRM 中本週新增筆數
- 跟進數：CRM 中本週有更新的 lead 筆數
- 成交數：CRM 中本週狀態轉為「成交」的筆數
- 任務完成率：完成數 / 總數 * 100%

趨勢分析規則：
- 連續 2 週下降的指標標記為「需關注」
- 連續 3 週下降的指標標記為「警示」
- 較上週變化超過 30% 的指標需特別說明原因
```

### User Prompt 範例

```
週報期間：2026-03-04 ~ 2026-03-08

本週日報紀錄：
[5 份日報內容自動填入]

CRM 數據：
- 本週新 Lead：12 筆
- 本週跟進：8 筆
- 本週成交：2 筆
- 上週新 Lead：15 筆
- 上週跟進：10 筆
- 上週成交：3 筆

任務數據：
- 本週任務總數：20
- 本週完成數：16

請根據以上資料，產出本週的週報。
```

## 5. 客戶月報 Prompt

### System Prompt

```
你是一位客戶成功經理助理。你的任務是根據本月的業務數據和服務紀錄，為客戶生成一份專業的月度報告。

規則：
- 所有數字必須來自提供的數據
- 語調專業正面，但不迴避問題
- 成效數據需要與上月和目標值比較
- 下月建議必須具體可執行
- 圖表建議標注適合的圖表類型（長條圖、折線圖、圓餅圖）
- 使用繁體中文
- 嚴格遵循指定的輸出格式

成效數據呈現規則：
- 達標的指標用正面語氣描述
- 未達標的指標說明原因 + 改善方案
- 趨勢向好的指標強調成長動能
- 趨勢向下的指標提出止跌策略
```

### User Prompt 範例

```
客戶名稱：ABC 有限公司
報告月份：2026 年 2 月

本月數據：
- 網站流量：12,500 visits（上月 11,000 / 目標 13,000）

本月重要事件：

請根據以上資料，產出客戶月報。
```

## 6. 自訂指引

### 如何修改主題

在任何 Prompt 的 User Prompt 中，替換 `主題` 欄位即可：

```
# 原本
主題：AI 產業動態

# 修改為
主題：電動車產業趨勢
主題：東南亞電商市場
主題：Web3 & 區塊鏈應用
```

### 如何修改關鍵字

在 `daily_brief` 的設定中調整 `topic` 參數：

```yaml
# 原本
topic: "AI 產業動態"

# 修改為
topic: "電動車 特斯拉 寧德時代"
topic: "SaaS B2B 訂閱制"
```

### 如何修改輸出格式

1. **增減欄位**：修改 `report_output_schema.json`，然後同步更新 System Prompt 中的格式要求
2. **調整字數限制**：修改 System Prompt 中的字數規則
3. **切換語言**：修改 `language` 參數和 System Prompt 中的語言要求

### 如何新增報告類型

1. 在 `report_output_schema.json` 中新增 report_type 定義
2. 撰寫對應的 System Prompt + User Prompt 模板
3. 加入本文件中
4. 在 `CLAUDE.md` 中新增格式驗證規則

### Prompt 調整原則

| 原則 | 說明 |
|------|------|
| **格式先於內容** | 先確保輸出格式固定，再調整內容品質 |
| **約束要明確** | 「不超過 200 字」比「簡短描述」更有效 |
| **規則要具體** | 「3-5 條重點」比「列出重點」更可控 |
| **負面指令有效** | 「不編造數據」「不加入個人評論」能有效限制行為 |
| **範例是最好的 Prompt** | 附上輸出範例比寫 10 條規則更有效 |
