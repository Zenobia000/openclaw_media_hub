# Notion CRM 資料庫模板規格

## 基本資訊

| 欄位 | 值 |
|------|-----|
| Database 名稱 | 客戶管理 CRM |
| 模組歸屬 | module_03_crm |
| 用途 | 集中管理潛在客戶、追蹤銷售進度、記錄 AI 互動歷程 |

---

## 欄位定義

| # | 欄位名稱 | Property Type | 選項 / 格式 | 說明 |
|---|---------|--------------|-------------|------|
| 1 | 客戶名稱 | Title | — | 主欄位，每筆紀錄的識別名稱 |
| 2 | 聯絡方式 | Rich Text | — | Email / Phone / Telegram handle，可填多個 |
| 3 | 來源管道 | Select | `Telegram` / `Email` / `表單` / `推薦` / `其他` | 客戶從哪裡來的 |
| 4 | 需求摘要 | Rich Text | — | AI 生成的 2-3 句需求摘要 |
| 5 | 狀態 | Select | `新詢問` / `跟進中` / `已報價` / `已成交` / `已流失` | 銷售漏斗階段 |
| 6 | 優先級 | Select | `高` / `中` / `低` | 根據 lead score 自動標註或人工調整 |
| 7 | 負責人 | Person | — | 指派負責跟進的團隊成員 |
| 8 | 詢問日期 | Date | YYYY-MM-DD | 首次詢問的日期 |
| 9 | Follow-up 日期 | Date | YYYY-MM-DD | 下次應跟進的日期 |
| 10 | 備註 | Rich Text | — | 人工補充的備註資訊 |
| 11 | AI 互動記錄 | Rich Text | — | AI 自動填寫，記錄每次互動摘要與時間戳 |

## 欄位設計說明

### 狀態欄位 — 銷售漏斗階段

```
新詢問 → 跟進中 → 已報價 → 已成交
                           ↘ 已流失
```

| 狀態 | 顏色建議 | 定義 |
|------|---------|------|
| 新詢問 | 藍色 | 剛收到的詢問，尚未聯繫 |
| 跟進中 | 黃色 | 已開始溝通，進行中 |
| 已報價 | 橙色 | 已發送報價或提案 |
| 已成交 | 綠色 | 成功簽約或付款 |
| 已流失 | 灰色 | 確認不會合作（可標記原因） |

### 來源管道欄位

| 選項 | 說明 |
|------|------|
| Telegram | 透過 Telegram Bot 對話進來 |
| Email | 透過 Email 詢問 |
| 表單 | 透過 Google Form / Typeform 等填寫 |
| 推薦 | 經由現有客戶或夥伴推薦 |
| 其他 | 以上皆非（建議在備註補充說明） |

### AI 互動記錄欄位格式

每次 AI 互動自動追加一筆，格式如下：

```
---
[2026-03-10 14:30] 來源：Telegram
摘要：客戶詢問企業內訓方案，團隊 15 人，預計 Q2 執行
評分：高
動作：已建立紀錄，follow-up 設為 2026-03-11
---
[2026-03-12 09:15] 來源：Telegram
摘要：客戶回覆確認預算範圍 5-8 萬，希望收到正式提案
評分：高
動作：狀態更新為「跟進中」，通知負責人準備提案
---
```

## View 設定建議

### 1. 看板視圖（Board View）— 主要工作視圖

- **分組依據**：狀態
- **排序**：Follow-up 日期（最近的在上面）
- **顯示欄位**：客戶名稱、需求摘要（前 50 字）、優先級、Follow-up 日期
- **用途**：一眼掌握各階段的客戶數量與分佈，拖拉更新狀態

### 2. 表格視圖（Table View）— 全覽視圖

- **顯示所有欄位**
- **排序**：詢問日期（最新在上）
- **篩選**：可快速切換「僅顯示進行中」「僅顯示本月」
- **用途**：資料總覽、匯出、批次編輯

### 3. 日曆視圖（Calendar View）— 排程視圖

- **日期依據**：Follow-up 日期
- **顯示**：客戶名稱 + 優先級標籤
- **用途**：規劃每日跟進工作，避免遺漏 follow-up

### 4. 篩選視圖（建議額外建立）

| 視圖名稱 | 篩選條件 | 用途 |
|----------|---------|------|
| 今日待跟進 | Follow-up 日期 = 今天 | 每日工作清單 |
| 高優先級 | 優先級 = 高 AND 狀態 ≠ 已成交/已流失 | 重點客戶追蹤 |
| 本月新詢問 | 詢問日期 = 本月 AND 狀態 = 新詢問 | 月度進件統計 |

## Notion MCP 連接說明

### 使用官方 Hosted MCP Server

Notion 提供官方的 MCP server，不需要自行部署，透過 OAuth 授權即可連接。

### 授權流程

```
1. 在 OpenClaw 設定中新增 Notion MCP 連接
2. 系統導向 Notion OAuth 授權頁面
3. 選擇要授權的 Workspace
4. 選擇要授權的 Database（選擇「客戶管理 CRM」）
5. 確認授權，完成連接
```

### 權限設定

| 權限 | 說明 | 是否需要 |
|------|------|---------|
| `read_content` | 讀取 Database 內容 | 是 |
| `insert_content` | 新增紀錄 | 是 |
| `update_content` | 更新既有紀錄 | 是（僅限狀態、AI 互動記錄欄位） |
| `delete_content` | 刪除紀錄 | 否（不授權） |

### MCP 操作範例

**新增紀錄**：
```
使用 Notion MCP 的 create page 功能：
- Database: 客戶管理 CRM
- Properties:
    客戶名稱: "王大明"
    聯絡方式: "wang@example.com"
    來源管道: "Telegram"
    需求摘要: "詢問企業內訓方案，15 人團隊，Q2 執行"
    狀態: "新詢問"
    優先級: "高"
    詢問日期: "2026-03-10"
    Follow-up 日期: "2026-03-11"
```

**更新 AI 互動記錄**：
```
使用 Notion MCP 的 update page 功能：
- Page ID: [existing_page_id]
- 追加到 AI 互動記錄欄位（append, 不覆寫）
```

## 建立步驟

1. 在 Notion Workspace 中建立新 Database
2. 按照欄位定義表建立所有 Property
3. 設定 Select 欄位的選項與顏色
4. 建立看板、表格、日曆三個 View
5. 透過 Notion MCP OAuth 授權連接到 OpenClaw
6. 用 `lead_capture` Skill 發送測試紀錄確認寫入正確

---

## 課堂提示

- Notion 免費版足以應付小型團隊的 CRM 需求，不需要升級付費版
- Database 建好後先手動新增 3-5 筆測試資料，確認欄位設計合理再接 AI
- 看板視圖是日常工作的主力，建議設為預設視圖
- 欄位不要一開始就加太多，先用這 11 個跑起來，後續再依需求擴充
