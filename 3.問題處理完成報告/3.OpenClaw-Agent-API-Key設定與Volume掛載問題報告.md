# OpenClaw Agent API Key 設定與 Volume 掛載問題報告

## 1. 問題現象與根因 (Phenomenon & Cause)

* **問題現象：** Agent 啟動時報錯，訊息為：
  ```
  No API key found for provider "anthropic".
  Auth store: /home/node/.openclaw/agents/main/agent/auth-profiles.json
  ```
  重啟容器後問題依舊。

* **發生原因：** 兩個核心問題疊加：
  1. **Volume 掛載路徑無效：** `docker-compose.yml` 預設將 `~/.openclaw`（Windows 使用者家目錄）掛載至容器內 `/home/node/.openclaw`，但該路徑在 Windows 環境下實際不存在或無法正確解析，導致容器內 `/home/node/.openclaw/` 為空。
  2. **缺少 `auth-profiles.json`：** OpenClaw Agent 不透過環境變數 `ANTHROPIC_API_KEY` 取得金鑰，而是讀取 `/home/node/.openclaw/agents/main/agent/auth-profiles.json`。該檔案從未被建立。

## 2. 解決方案 (Resolution)

* **處理步驟：**
  1. **修改 `docker-compose.yml` 的 Volume 掛載路徑：** 將兩個 service（`openclaw-gateway` 與 `openclaw-cli`）的 volume 預設值從 `~/.openclaw` 改為 `./.openclaw`（專案目錄內），確保路徑可被正確解析。
     ```yaml
     # 修改前
     - ${OPENCLAW_CONFIG_DIR:-~/.openclaw}:/home/node/.openclaw
     - ${OPENCLAW_WORKSPACE_DIR:-~/.openclaw/workspace}:/home/node/.openclaw/workspace

     # 修改後
     - ${OPENCLAW_CONFIG_DIR:-./.openclaw}:/home/node/.openclaw
     - ${OPENCLAW_WORKSPACE_DIR:-./.openclaw/workspace}:/home/node/.openclaw/workspace
     ```
  2. **在專案內建立 `.openclaw` 目錄結構：** 建立 `D:\openclaw\.openclaw\` 及其子目錄。
  3. **建立 `openclaw.json`：** 將 Gateway 設定檔放置於 `.openclaw/openclaw.json`，包含 gateway mode、auth token、controlUi 等設定。
  4. **建立 `auth-profiles.json`：** 將 `.env` 中的 Anthropic API Key 寫入 `.openclaw/agents/main/agent/auth-profiles.json`，格式如下：
     ```json
     {
       "anthropic": {
         "apiKey": "sk-ant-..."
       }
     }
     ```
  5. **重啟容器：** 執行 `docker compose down && docker compose up -d`。

* **最終結果：** Volume 掛載指向專案內 `.openclaw` 資料夾，容器可正確讀取 `auth-profiles.json` 中的 Anthropic API Key。

## 3. 常用指令與操作 (Commands & Operations)

* **指令一：** `docker compose down && docker compose up -d`
  * **用途：** 完整重啟所有容器，使 volume 掛載與設定變更生效。

* **指令二：** `MSYS_NO_PATHCONV=1 docker exec openclaw-gateway ls -la /home/node/.openclaw/`
  * **用途：** 在 Git Bash 環境下檢查容器內檔案是否正確掛載（`MSYS_NO_PATHCONV=1` 防止路徑被自動轉換）。

* **指令三：** `docker exec openclaw-gateway cat /home/node/.openclaw/agents/main/agent/auth-profiles.json`
  * **用途：** 驗證容器內 Agent 的 API Key 設定檔內容。

## 4. 結論 (Conclusion)

在 Windows + Docker 環境下，`~` 路徑展開可能不如預期，應優先使用專案相對路徑（`./.openclaw`）作為 volume 掛載來源，確保設定檔可被容器正確存取。OpenClaw Agent 的 API Key 需透過 `auth-profiles.json` 設定，而非環境變數。
