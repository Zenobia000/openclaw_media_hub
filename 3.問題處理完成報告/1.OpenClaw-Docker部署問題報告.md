# 1. OpenClaw Docker 部署 — 問題處理完成報告

**日期：** 2026-03-15
**環境：** Windows 11 Pro / Docker Desktop / Docker Compose v2

---

## 目標

根據 `openclaw-Docker.md` 文件，建立 `docker-compose.yml` 並成功啟動 OpenClaw 網關。

---

## 遭遇問題與解決方式

### 問題 1：`version` 屬性已過時

**現象：** Docker Compose 警告 `the attribute 'version' is obsolete`
**原因：** 現代 Docker Compose v2 已不再需要 `version` 欄位。
**解決：** 移除 `docker-compose.yml` 中的 `version: "3.8"`。

---

### 問題 2：`unknown option '--allow-unconfigured'`

**現象：** 網關容器啟動後立即報錯退出。
**原因：** 遠端映像 `ghcr.io/openclaw/openclaw:latest` 的入口點不接受 `--allow-unconfigured` 作為直接參數。
**解決：** 將 `command` 改為 `["openclaw", "gateway"]`，透過 CLI 子指令啟動網關。

---

### 問題 3：`Missing config` 導致容器不斷重啟

**現象：** 日誌顯示 `Missing config. Run 'openclaw setup' or set gateway.mode=local`。
**原因：** 首次啟動時尚未進行初始設定，缺少 `gateway.mode` 配置。
**解決：** 先停止網關，用獨立容器執行設定指令：

```bash
docker run --rm --entrypoint "" \
  -v "${USERPROFILE}/.openclaw:/home/node/.openclaw" \
  ghcr.io/openclaw/openclaw:latest \
  openclaw config set gateway.mode local
```

---

### 問題 4：`pull access denied for openclaw`

**現象：** 未設定 `OPENCLAW_IMAGE` 時，Compose 嘗試拉取不存在的 `openclaw:local` 映像。
**原因：** `docker-compose.yml` 預設值 `${OPENCLAW_IMAGE:-openclaw:local}` 在未設定環境變數時指向本地不存在的映像。
**解決：** 在專案根目錄建立 `.env` 檔案：

```
OPENCLAW_IMAGE=ghcr.io/openclaw/openclaw:latest
```

---

### 問題 5：主機無法連線 `ERR_EMPTY_RESPONSE`

**現象：** 瀏覽器開啟 `http://127.0.0.1:18789/` 顯示「未傳送任何資料」。
**原因：** 容器內網關綁定在 `127.0.0.1`，此地址僅在容器網路命名空間內有效。Docker Desktop for Windows 透過連接埠轉發連線時，需要容器內服務綁定到 `0.0.0.0` 才能從主機存取。
**驗證：**

- `docker exec` 在容器內部 `curl healthz` → 成功
- 主機直接 `curl healthz` → 失敗（Empty reply）

**解決：** 將網關綁定模式改為 `lan`（即 `0.0.0.0`）：

```bash
docker run --rm --entrypoint "" \
  -v "${USERPROFILE}/.openclaw:/home/node/.openclaw" \
  ghcr.io/openclaw/openclaw:latest \
  openclaw config set gateway.bind lan
```

重啟後網關綁定 `ws://0.0.0.0:18789`，主機可正常連線。

## 常用指令

| 操作               | 指令                                                          |
| ------------------ | ------------------------------------------------------------- |
| 啟動網關           | `docker compose up -d openclaw-gateway`                       |
| 取得 Dashboard URL | `docker compose run -T --rm openclaw-cli dashboard --no-open` |
| 健康檢查           | `curl -fsS http://127.0.0.1:18789/healthz`                    |
| 查看日誌           | `docker logs openclaw-gateway`                                |
| 停止網關           | `docker compose stop openclaw-gateway`                        |

---

## 結論

所有問題均已解決，OpenClaw 網關已在 Docker 中穩定運行，Dashboard 可透過瀏覽器正常存取。核心問題在於 Docker Desktop for Windows 環境下，容器內服務必須綁定 `0.0.0.0` 而非 `127.0.0.1`，才能透過連接埠轉發從主機存取。
