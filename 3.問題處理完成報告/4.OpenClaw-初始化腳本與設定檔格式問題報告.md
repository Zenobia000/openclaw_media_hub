# OpenClaw 初始化腳本與設定檔格式問題報告

## 1. 問題現象與根因 (Phenomenon & Cause)

* **問題現象：** 手動建立 `.openclaw/openclaw.json` 設定檔後，容器啟動時連續出現兩類錯誤：
  1. `Invalid config` — `gateway.mode: Invalid input (allowed: "local", "remote")` 及 `gateway.controlUi: Invalid input: expected object, received boolean`。
  2. `Missing config` — `Run openclaw setup or set gateway.mode=local (or pass --allow-unconfigured)`。

* **發生原因：** 兩個問題疊加：
  1. **設定值不符合 Schema：** `gateway.mode` 被設為 `"gateway"`，但 OpenClaw 僅接受 `"local"` 或 `"remote"`；`gateway.controlUi` 被設為布林值 `true`，但 Schema 要求為物件型別。
  2. **初始化腳本環境不相容：** 原始腳本以 Bash（`.sh`）撰寫，無法在 Windows 環境直接執行，需改為 PowerShell（`.ps1`）。

## 2. 解決方案 (Resolution)

* **處理步驟：**
  1. **將初始化腳本從 Bash 改寫為 PowerShell：** 刪除 `init-openclaw.sh`，建立 `init-openclaw.ps1`，確保在 Windows 環境可直接執行。
  2. **修正 `openclaw.json` 設定格式：** 將 `gateway.mode` 從 `"gateway"` 改為 `"local"`，移除不正確的 `controlUi: true` 欄位，僅保留最小化合法設定。
  3. **移除自動讀取 API Key 邏輯：** 腳本不再從 `.env` 讀取 `ANTHROPIC_API_KEY`，改為產生 `auth-profiles.json` 佔位範本，由使用者手動填入。
  4. **移除容器內 `openclaw doctor --fix` 自動修復步驟：** 因容器在缺少正確設定檔時無法啟動，該指令無法在啟動前執行，改為在腳本中直接產生合法設定檔。

* **最終結果：** `init-openclaw.ps1` 可在 Windows 環境正確執行，產生的 `.openclaw/openclaw.json` 符合 Schema 要求，容器可正常啟動。

## 3. 常用指令與操作 (Commands & Operations)

* **指令一：** `.\init-openclaw.ps1`
  * **用途：** 在專案目錄內建立 `.openclaw` 完整目錄結構、`openclaw.json` 設定檔及 `auth-profiles.json` 範本。

* **指令二：** `docker compose up -d`
  * **用途：** 初始化完成並填入 API Key 後，啟動所有容器。

* **指令三：** `docker exec openclaw-gateway openclaw doctor --fix`
  * **用途：** 容器啟動後，若仍有設定問題可在容器內執行此指令自動修復。

## 4. 結論 (Conclusion)

OpenClaw 的 `openclaw.json` 設定檔具有嚴格的 Schema 驗證，`gateway.mode` 僅接受 `"local"` 或 `"remote"`，欄位型別需完全匹配。初始化腳本應產生最小化且合法的設定，避免猜測未知欄位格式；Windows 環境下應優先使用 PowerShell 腳本確保相容性。
