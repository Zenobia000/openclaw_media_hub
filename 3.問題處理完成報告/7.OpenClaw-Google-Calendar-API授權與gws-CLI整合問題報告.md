# Google Calendar API 授權與 gws CLI 整合問題報告

## 1. 問題現象與根因 (Phenomenon & Cause)

* **問題現象：** 執行預約模組安裝（`/install-booking`）完成後，需設定 Google Calendar API 授權以啟用日曆事件建立功能。過程中遭遇三個主要障礙：
  1. OAuth 憑證類型錯誤 — 初始建立的憑證為「Web 應用程式」類型，無法用於本地 CLI 授權流程。
  2. `gws auth login` 授權失敗 — 瀏覽器顯示 `400 invalid_request: Required parameter is missing: response_type` 錯誤。
  3. `gws auth setup` 帳號不匹配 — 自動建立的 OAuth 專案使用了與目標日曆帳號（ed32588x@gmail.com）不同的 Google 帳號。

* **發生原因：**
  1. **憑證類型錯誤：** Google Cloud Console 建立 OAuth 用戶端 ID 時選擇了「Web 應用程式」，產出 JSON 格式為 `{"web": {...}}`。而 `google-calendar-mcp` 及 `gws` CLI 均要求「桌面應用程式（Desktop app）」類型，JSON 格式需為 `{"installed": {...}}`。
  2. **OAuth 400 錯誤：** OAuth 同意畫面處於「測試模式」，且未將目標帳號 ed32588x@gmail.com 加入測試使用者名單，導致 Google 拒絕授權請求。
  3. **帳號不匹配：** `gws auth setup` 會以當前瀏覽器預設登入帳號建立 OAuth 專案，若該帳號非目標日曆帳號則導致授權對象錯誤。

## 2. 解決方案 (Resolution)

* **處理步驟：**
  1. 在 Google Cloud Console 重新建立 OAuth 用戶端 ID，選擇「**電腦應用程式（Desktop app）**」類型，下載新的 JSON 憑證檔。
  2. 將憑證檔複製至預約模組標準路徑：`module_pack/module_01_booking/gcp-oauth.keys.json`。
  3. 更新 `.gitignore`，排除 `gcp-oauth.keys.json`、`client_secret_*.json`、`.google-calendar-mcp-token.json` 等機密檔案。
  4. 透過 `@cocal/google-calendar-mcp auth` 完成 OAuth 授權流程，成功取得 Token（儲存於 `C:\Users\x1596\.config\google-calendar-mcp\tokens.json`）。
  5. 安裝 Google Workspace CLI（`@googleworkspace/cli`），將已授權的 Token 轉換為 `gws` CLI 所需的 `credentials.json` 格式（需包含 `type`、`client_id`、`client_secret`、`access_token`、`refresh_token` 欄位）。
  6. 在 Google Cloud Console OAuth 同意畫面中，將 ed32588x@gmail.com 加入測試使用者。
  7. 使用 `gws calendar events insert` 成功建立測試事件，驗證整合完成。

* **最終結果：** Google Calendar API 授權設定完成。透過 `gws` CLI 成功在 ed32588x@gmail.com 的 Google Calendar 上建立預約事件「桑尼工作室 - 測試預約」（2026-03-18 14:00-15:00），狀態為 `confirmed`。

## 3. 常用指令與操作 (Commands & Operations)

* **指令一：** `npx @googleworkspace/cli auth status`
  * **用途：** 檢查 gws CLI 的授權狀態，確認憑證路徑與認證方式。

* **指令二：** `npx @googleworkspace/cli calendar events list --params '{"calendarId":"primary","maxResults":5,"timeMin":"2026-03-17T00:00:00+08:00","singleEvents":true,"orderBy":"startTime"}'`
  * **用途：** 查詢 Google Calendar 上的近期事件，用於驗證連線與授權是否正常。

* **指令三：** `npx @googleworkspace/cli calendar events insert --params "{\"calendarId\":\"primary\"}" --json "$EVENT_JSON"`
  * **用途：** 建立 Google Calendar 事件。Windows 環境下需將 JSON body 存入變數以單行傳入，避免多行 JSON 解析失敗。

* **指令四：** `GOOGLE_OAUTH_CREDENTIALS="/path/to/gcp-oauth.keys.json" npx -y @cocal/google-calendar-mcp auth`
  * **用途：** 執行 google-calendar-mcp 的 OAuth 授權流程，取得 refresh token。

* **指令五：** `claude mcp add google-calendar -s project -e GOOGLE_OAUTH_CREDENTIALS="path" -- npx -y @cocal/google-calendar-mcp`
  * **用途：** 將 google-calendar MCP Server 加入 Claude Code 專案設定（`.mcp.json`）。

## 4. 結論 (Conclusion)

Google Calendar API 整合需注意三個關鍵點：(1) OAuth 用戶端類型必須選擇「桌面應用程式」而非「Web 應用程式」；(2) 測試模式下須手動將目標帳號加入 OAuth 同意畫面的測試使用者；(3) Windows 環境下 `gws` CLI 的 `--json` 參數不支援多行 JSON，需以單行變數或檔案方式傳入。建議將 `2.金鑰取得/3.Google/文件.md` 教學指引中的憑證類型說明從「Web 應用程式」修正為「電腦應用程式」，以避免學員重複踩坑。
