# Part A — 課程核心

---

## 1. 課程核心理念

V1 教「從零開始做」，V2 教「裝好就改」。

這堂課賣的不是「OpenClaw 很厲害」或「Claude Code 可以寫程式」。
賣的是：

> **一套 AI 助理安裝包，幫個體戶和小公司把預約、Email、CRM、客服、報告、通知，先跑起來。**

學員不是來學開發的，是來裝系統的。
上課不是從零寫，是選模組 → 填設定表 → Claude Code 自動改模板 → 跑起來。

**核心主張：** 先模板後客製 / 先 Draft 後 Auto-send / 先單入口後多通道。

**目標：** 學員在一天結束時，擁有一個 Telegram AI 助理入口 + 至少 3 個已運作的商務模組 + 一份自己的商業設定檔 + 一份故障排除 SOP。不需要 Claude Pro/Max 訂閱即可參加——可選用 OpenRouter 免費模型（零成本）或 Claude 訂閱（最佳效能）。

## 2. 核心框架

### 模組化安裝包架構

從「學寫程式」到「裝模組改設定」的思維轉換：

| 層級 | 做什麼 | 工具 | 人的角色 |
|------|--------|------|----------|
| L1 | 裝模組 | `/install-xxx` | 選擇者：選需要的模組 |
| L2 | 填設定 | business_profile_form | 定義者：填商業欄位 |
| L3 | AI 改模板 | Claude Code + CLAUDE.md | 監督者：審查 AI 的修改 |
| L4 | 跑起來 | OpenClaw + Docker | 使用者：在手機上測試 |

```
L4  🤖 OpenClaw 跑起來 → AI 24/7 自動工作
L3  🔧 Claude Code 改模板 → Vibe Coding 客製化
L2  📋 填設定表 → 商業欄位，不是技術參數
L1  📦 選模組安裝 → /install 一鍵搞定
```

**口訣：** 選了就裝，裝了就填，填了就改，改了就跑。

### 系統角色分層

| 工具 | 角色 | 負責什麼 |
|------|------|----------|
| Claude Code | Vibe Coding 引擎 | 改模板、生設定、修 Skill、改 SOP |
| OpenClaw | 聊天入口 + 任務編排 | 接收對話、分派任務、排程執行 |
| Notion / Google Sheets | CRM / 記錄面 | 儲存客戶資料、任務紀錄、報告 |
| Google Calendar | 預約工具 | 查空檔、建事件、發提醒 |
| Gmail | 讀信工具 | 讀信分類、標籤管理 |
| Resend | 發信工具 | 通知信、回覆信、行銷信（100 封/天免費） |
| Telegram | AI 助理入口 | 單一對話介面，24/7 運作 |

## 3. 六大模組總覽

| # | 模組 | 目的 | 最小可行流程 | 課堂版重點 |
|---|------|------|-------------|-----------|
| 1 | 預約模組 | AI 查空檔、建行程、發確認 | 對話 → 解析時間 → Calendar 事件 → 確認訊息 → CRM 紀錄 | 只做建立+確認，不做改期取消 |
| 2 | Email 模組 | AI 讀信分類、草擬回覆 | 收信 → 摘要 → 分類 → 草擬回信 → 人工確認送出 | Draft-first，不自動送信 |
| 3 | CRM 模組 | 把詢問收斂成可追蹤的 pipeline | 客戶提問 → AI 摘要 → 寫入 CRM → 標狀態 → 建 follow-up | 只選一個系統（Notion 或 Sheets） |
| 4 | 客服模組 | AI 回常見問題，超限轉人工 | 提問 → FAQ 回答 → 判斷升級 → CRM 紀錄 → 通知人工 | 單一 Telegram 入口，DM 模式 |
| 5 | 報告模組 | 定時查資料、整理、產報告 | 定時觸發 → 查資料 → 摘要 → 固定格式輸出 → 寫入/通知 | 固定輸出格式，三種題材擇一 |
| 6 | 通知模組 | 系統主動提醒，不只被動對話 | 排程到點 → 執行任務 → 送通知 → 寫紀錄 | 每天跑一次，不做複雜排程 |

## 4. 目標受眾 (ICP)

### 主要族群（V2 更聚焦商務需求）
- **個體戶 / 一人公司：** 自己接案、自己回客戶、自己做行政，需要 AI 分擔日常工作
- **小團隊創業者：** 3-10 人公司，沒有 IT 部門，想用最低成本建立自動化流程
- **自由工作者 / 顧問：** 需要預約管理、客戶追蹤、定期報告，但不想花時間在行政上

### 次要族群
- **行銷/PM/業務：** 想自動化客戶管理、報告產出、訊息通知
- **AI 工具愛好者：** 已用過 ChatGPT / Claude，想讓 AI 真的「做事」不只「聊天」

### 學員畫像
- 每天花 2+ 小時在回訊息、排預約、整理資料、寫報告上
- 聽過 AI 自動化但不知道怎麼落地
- 不會寫程式，但願意花一天學會「裝系統」
- 想要的是「今天回去就能用的東西」，不是理論

## 5. 課程大綱 (8 小時)

### 第一單元 (09:00 - 10:20): 系統概覽與環境建置

> 先看全貌，再裝引擎。

**主題：** 理解模組化安裝包的設計哲學，完成環境建置，啟動 OpenClaw。

**目標：** 學員理解六大模組、完成所有環境設定、成功啟動 OpenClaw + Telegram Bot。

**內容：**

1. **觀念破冰：你不是來學開發的 (15 min)**
   - 從「學寫程式」到「裝模組改設定」的思維轉換
   - 六大模組展示：預覽今天會裝好的東西（現場 Demo 完成品）
   - 系統角色分層：誰負責什麼（Claude Code / OpenClaw / Notion / Google）
   - 五大安全原則：先 Draft 後 Auto-send、最小權限、白名單

2. **環境建置 (30 min)**
   - 終端機入門（三個救命指令：ls / cd / pwd）
   - 安裝 Claude Code + 帳號綁定
     - 方案 A：Claude Pro/Max 訂閱（最佳效能，直接登入即可）
     - 方案 B：OpenRouter 免費模型（零成本，設定 ANTHROPIC_BASE_URL 環境變數指向 OpenRouter，使用 qwen3-coder:free / deepseek-r1:free 等免費模型，需設定 Haiku/Sonnet/Opus 三個模型層級）
     - 現場示範兩種方案的設定，學員擇一完成
   - Docker Desktop 確認 + Docker Compose 基本操作
   - Google API 授權設定（Calendar + Gmail scopes）

3. **啟動 OpenClaw + Telegram Bot (30 min)**
   - BotFather 建立 Telegram Bot
   - docker-compose 一鍵啟動 OpenClaw
   - 連接 Telegram → 測試第一句對話
   - System Prompt 人格設計：你的 AI 助理叫什麼名字？什麼口吻？

**實作活動：** 啟動 OpenClaw，在手機上跟自己的 Telegram AI 助理說話。

### 第二單元 (10:30 - 12:00): 商業設定 + 預約模組 + Email 模組

> 填表格，不寫程式。AI 幫你改好一切。

**主題：** 填寫商業設定表、安裝並客製化預約模組與 Email 模組。

**目標：** 學員完成商業設定、預約模組可在 Telegram 上接受預約、Email 模組可讀信分類。

**內容：**

1. **商業設定表：你的 AI 助理長什麼樣 (20 min)**
   - 填寫 `business_profile_form`：公司名、口吻、時段、簽名檔、FAQ
   - 用 Claude Code 讀取設定表，自動修改所有模板
   - Vibe Coding 的核心：你描述需求，AI 改技術設定

2. **模組 1 安裝：預約模組 (25 min)**
   - `/install-booking`：安裝 calendar_booking_skill
   - 修改 `calendar_fields.json`：可預約時段、時長、事件標題格式
   - 測試：在 Telegram 上說「我想約下週三下午」→ 看 Calendar 出現事件
   - 最佳實務：第一版只做建立+確認，不做改期取消

3. **模組 2 安裝：Email 模組 (30 min)**
   - `/install-email`：安裝兩個 provider——Resend（發信）+ Gmail API 或 n8n（讀信）
   - **發信：Resend**（設定 API key + MCP Server，一分鐘搞定）
     - 註冊 Resend → 取得 API key → 設定 MCP Server → 完成
     - 免費額度：100 封/天、3,000 封/月，對個體戶綽綽有餘
   - **讀信：Gmail API（進階）或 n8n Gmail node（簡化版）**
     - Gmail API：完整控制，但 OAuth 設定較複雜（7-8 步）
     - n8n Gmail node：圖形化設定，適合怕指令的學員
   - 修改 `reply_style_claude.md`：回信口吻、簽名檔、分類規則
   - 測試：用 Resend 發一封測試信 → 用 Gmail/n8n 讀信分類 → 看分類結果
   - **鐵律：Draft-first。** 不自動送信。人工確認後才送出。
   - 注意：Resend 不能讀 Gmail 信箱，讀信分類仍需 Gmail API 或 n8n

4. **Debug 時間 + Q&A (15 min)**
   - 常見卡點排除
   - Claude Code 就是最好的除錯工具：「幫我看看為什麼預約沒建立」

**實作活動：** 在 Telegram 上測試預約流程 + 觸發一次 Email 分類。

### 第三單元 (13:00 - 14:30): CRM 模組 + 客服模組

> 客人來了，AI 幫你記住每一個。

**主題：** 安裝 CRM 模組建立客戶追蹤管道，安裝客服模組讓 AI 自動回常見問題。

**目標：** CRM 可自動記錄客戶資料、客服可自動回答 FAQ 並轉人工。

**內容：**

1. **模組 3 安裝：CRM 模組 (30 min)**
   - 選擇主資料面：Notion 或 Google Sheets（不要兩個一起用）
   - Notion 路線：`/install-crm` → 建立 CRM Database → 連接 Notion MCP
   - Sheets 路線：`/install-crm` → 複製 Sheets 模板 → 設定 API
   - 修改 `lead_score_rules.md`：評分規則依自身業務調整
   - 測試：在 Telegram 上模擬客戶詢問 → 看 CRM 自動新增紀錄

2. **模組 4 安裝：客服模組 (30 min)**
   - `/install-customer-service`：安裝 faq_responder_skill
   - 載入自己的 FAQ（從商業設定表填的那份）
   - 設定升級規則：什麼問題 AI 回、什麼問題轉人工
   - 測試：問 FAQ 內的問題 → AI 回答
   - 測試：問報價 → AI 說「讓我幫您轉接專人」→ 你收到通知
   - **原則：客服不是越自動越好。該自動的自動，該人工的人工。**

3. **串連測試：CRM + 客服 (20 min)**
   - 客戶在 Telegram 問問題 → AI 回答 + CRM 自動紀錄
   - 客戶問報價 → AI 轉人工 + CRM 標為「需人工處理」
   - 看看 Notion/Sheets 裡面的數據有沒有正確寫入

4. **Debug 時間 (10 min)**

**實作活動：** 請隔壁學員用 Telegram 當客戶問你問題，看 CRM 是否正確紀錄。

### 第四單元 (14:40 - 16:00): 報告模組 + 通知模組

> AI 不只被動回答，它會主動幫你做事。

**主題：** 安裝報告模組讓 AI 自動產報告，安裝通知模組讓系統主動提醒。

**目標：** 報告可手動觸發產出、通知可定時發送。

**內容：**

1. **模組 5 安裝：報告模組 (30 min)**
   - `/install-report`：安裝 daily_brief_skill + weekly_report_skill
   - 選報告題材（三選一）：每日新聞摘要 / 產業觀察 / 專案日報
   - 修改 `report_prompt_pack.md`：主題、關鍵字、格式
   - 測試：手動觸發 → 看報告產出 → 確認格式是否固定
   - **關鍵：固定輸出格式。** 不讓每次報告長得不一樣。

2. **模組 6 安裝：通知模組 (25 min)**
   - `/install-notify`：安裝 scheduled_notify_skill
   - 設定排程：每天幾點跑報告、幾點做 CRM 提醒
   - 設定通知規則：日常走 Telegram、異常走 Telegram + Email
   - 測試：手動觸發一次排程 → 看通知有沒有收到
   - **限制說明：** 為什麼第一版不做複雜排程（OpenClaw cron 已知問題）

3. **全鏈路串連 Demo (15 min)**
   - 從頭走一次：客戶在 Telegram 問問題 → AI 回答 → 記 CRM → 到了晚上 → AI 跑日報 → 寄通知
   - 看 CRM 裡一天累積的資料
   - 這就是「一台裝好的小機器」

4. **Debug 時間 (10 min)**

**實作活動：** 手動觸發一次日報，在 Telegram 收到報告通知。

### 第五單元 (16:10 - 17:10): 客製化 + 成果展示

> 從「裝好」到「變成你的」。

**主題：** 用 Vibe Coding 深度客製化、成果展示與互評。

**內容：**

1. **深度客製化工作坊 (25 min)**
   - 用 Claude Code 修改任何你想改的東西：
     - 「把客服口吻改成更正式」
     - 「在 CRM 加一個『預算範圍』欄位」
     - 「把日報格式改成包含今日待辦」
     - 「預約確認訊息加上 Google Maps 連結」
   - 自由改、自由試，講師和助教巡場協助
   - 這就是 Vibe Coding：用自然語言描述需求，AI 改技術設定

2. **故障排除 SOP 說明 (10 min)**
   - 發放 `troubleshooting_sop.md`

   - 回家後壞了怎麼辦：先查 SOP → 問社群 → 聯繫講師

3. **成果展示與互評 (20 min)**
   - 每位學員展示自己的 AI 助理（用手機展示 Telegram 對話）
   - 互相加對方的 Bot，實際體驗
   - 投票選出「最佳商務助理」和「最有創意獎」

4. **進階路線圖 (5 min)**
   - 安裝版 → 客製版 → 代維運版
   - 課後 30 天養成計畫
   - 進階工作坊預告：多代理系統、n8n 整合

## 11. 教學設計原則

### 1. 填表 > 寫程式
- 學員填的是商業欄位（公司名、口吻、時段），不是技術參數
- Claude Code 依設定表自動修改所有模板
- 最困難的操作是 copy-paste 和改幾個字

### 2. 每 30 分鐘一個可測試的成果
- 第一單元結束：Telegram Bot 會說話
- 第二單元結束：可以預約 + 讀信
- 第三單元結束：CRM 有資料 + 客服會回答
- 第四單元結束：報告會產出 + 通知會發送

### 3. 安全第一
- 所有模組預設最小權限
- Email 只 Draft、Calendar 只 Create、CRM 只 Append
- Skill 白名單制度，不自由安裝
- 人工確認節點：AI 草擬 → 人看過 → 才執行

### 4. 失敗友好
- 每個模組有「緊急救援包」：預設好的 docker-compose + 完整設定
- Claude Code 就是最好的除錯工具：「幫我看看為什麼 Email 分類不對」
- 故障排除 SOP 涵蓋所有已知卡點
- 助教比例 1:5

### 5. 一個入口原則
- 入口：Telegram（不同時教多平台）
- 資料面：Notion 或 Sheets（擇一，不並行）
- 通知面：Telegram 或 Email（擇一為主）

## 13. FAQ

### Q: 我完全不會寫程式，真的可以來嗎？
**A:** 可以。這堂課的核心是「填表格、裝模組」。所有技術操作由 Claude Code 代勞，你只需要知道自己想要什麼。

### Q: 跟 V1（Claude Code + OpenClaw 基礎版）有什麼不同？
**A:** V1 教你「用 AI 寫程式」，V2 教你「用 AI 自動化你的工作」。V2 直接給你六大商務模組，填設定就能用。如果你的目標是讓 AI 幫你處理日常工作，V2 更適合。

### Q: 我只是一個人的小公司，需要這麼多模組嗎？
**A:** 越是一個人，越需要。這六個模組（預約、Email、CRM、客服、報告、通知）就是一個人做不完的日常工作。上完課你等於多了一個 24/7 的數位助理。

### Q: OpenClaw 安全嗎？會不會 AI 亂寄信、亂回客戶？
**A:** 我們採用最嚴格的安全原則：Email 只存草稿不自動送、所有對外行為需人工確認、Skills 走白名單、整個系統在 Docker 隔離環境運行。

### Q: Claude Code 需要付費嗎？
**A:** 不一定。現在有兩種方案：(1) Claude Pro/Max 訂閱（$20/月起，效能最佳）；(2) OpenRouter 免費模型（零成本，設定環境變數即可使用 qwen3-coder:free、deepseek-r1:free 等模型）。課堂上會現場示範兩種方案，學員可自由選擇。

### Q: 用免費模型效果會差很多嗎？
**A:** 免費模型可以完成所有課堂練習，但在以下方面與付費模型有差異：(1) 回覆速度較慢（可能需等待 5-15 秒）；(2) 複雜任務的理解力略低；(3) 可能有 rate limit 限制（尖峰時段需等待）。建議先用免費方案上手，確認有長期使用需求後再考慮訂閱。

### Q: 費用大概多少？
**A:** 最低可以零成本起步：Claude Code 用 OpenRouter 免費模型、Email 用 Resend 免費額度（100 封/天）、其他工具（Google、Notion、Telegram）都是免費。若選擇付費方案：Claude Pro 訂閱 $20/月 + API 使用約 $3-10/月，總計約 NT$800-1,200/月。

### Q: 課程結束後，系統會消失嗎？
**A:** 不會。OpenClaw 跑在你自己的 Docker 裡、CRM 在你的 Notion/Sheets 帳號下、Telegram Bot 是你自己建的。全部是你的資產。

### Q: 如果回家後系統壞了怎麼辦？
**A:** 我們提供完整的故障排除 SOP、30 天 Telegram 群組支援、錄影回放。大多數問題在 SOP 裡都有解。

## 16. 風險識別與應對

### 1. 環境建置失敗
- **風險：** Google API 授權設定複雜、Docker 安裝卡關
- **應對：** 課前 3 天發送安裝指南 + Office Hour；Google API 預設好 OAuth 流程截圖包；GitHub Codespaces 備案

### 2. API 費用與額度
- **風險：** 學員 API 額度不足或超支；OpenRouter 免費模型可能遇到 rate limit（尖峰時段排隊等待）
- **應對：** 課前說明費用（建議儲值 $10）；教 Token 節省技巧；備用 API Key；免費模型遇 rate limit 時切換備用模型或使用講師提供的臨時 API Key

### 2.5. 免費模型效能限制
- **風險：** 免費模型回覆品質不如 Claude Opus/Sonnet，複雜任務可能失敗或需多次嘗試
- **應對：** 課堂模板已針對免費模型優化 Prompt；複雜操作由講師示範付費模型效果作為對照；提供 Prompt 簡化技巧降低模型負擔

### 3. OpenClaw 穩定性
- **風險：** OpenClaw 排程功能有已知 issue（cron 重複觸發、topic routing 問題）
- **應對：** 第一版避開複雜排程；使用手動觸發 + 簡單 cron；走 DM 不走 topic

### 4. Google API 授權複雜度
- **風險：** OAuth 流程對非技術用戶太困難
- **應對：** 講師預做好截圖式 step-by-step 教學；助教一對一協助；最小 scopes 降低授權步驟

### 5. 安全風險
- **風險：** 學員不當設定導致 AI 自動寄信、API Key 洩露
- **應對：** 強制 Draft-first 原則；Docker 隔離；Skills 白名單；claude code 保留權限提示

### 6. 進度落差
- **風險：** 部分學員跟不上六大模組的安裝節奏
- **應對：** 最低目標是完成 3 個模組（預約+CRM+客服）；每模組有「一鍵救援包」；快的學員有加速挑戰（深度客製化）

### 7. ClawHub 惡意 Skills
- **風險：** 公開 Skills 市集有惡意 Skill 風險（已有通報案例）
- **應對：** 白名單制度；只裝講師審過的 Skills；所有 Skill 有版本號備份
